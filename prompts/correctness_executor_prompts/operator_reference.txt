# STeP Operator Reference

## Shape Notation

All streams have shape `(1, d0, d1, ..., dN)` where the leading `1` is implicit.
Tile dtype is written as `Tile(rows, cols)`. Shape transformations show how
input shape maps to output shape.

---

## Data Movement Operators

### OffChipLoad — Load tiles from HBM
```python
OffChipLoad(
    underlying=torch.Tensor,       # Source tensor
    stride=(s0, s1, ...),          # Stride per output dimension (0 = broadcast)
    out_shape_tiled=(d0, d1, ...), # Output stream shape (in tiles)
    tile_row=int,                  # Tile height
    tile_col=int,                  # Tile width
    par_dispatch=int,              # Parallel tile transfers (tunable per-operator)
    mock_bf16=False,
)
```
**Shape rule:** Output stream shape = `(1,) + out_shape_tiled`, dtype = `Tile(tile_row, tile_col)`

**Stride convention:** `stride[i]` = number of tiles to advance in the underlying tensor when dimension `i` of `out_shape_tiled` increments by 1. Use `0` for broadcast dimensions (same data reused across that dimension).

**CRITICAL:** `underlying` must be **at least 2D**. The constructor indexes `underlying.shape[-2]` and `underlying.shape[-1]`. A 1D tensor (e.g. `torch.full((1,), ...)`) will cause `IndexError: tuple index out of range`. For scalars, use shape `(1, 1)`.

**Example — GEMM input loading:**
```python
# x is (M, K), tiled into (M//tm, N//tn, K//tk) for weight-stationary GEMM
# stride: M-dim advances by K//tk tiles, N-dim broadcast (0), K-dim advances by 1
x_load = OffChipLoad(
    underlying=x,
    stride=(K // tile_k, 0, 1),
    out_shape_tiled=(M // tile_m, N // tile_n, K // tile_k),
    tile_row=tile_m, tile_col=tile_k, par_dispatch=4,
)
# Stream: shape=(1, M//tm, N//tn, K//tk), dtype=Tile(tm, tk)
```

**Example — Scalar/constant loading (e.g. 1/sqrt(dim)):**
```python
# Scalar must be 2D: shape (1, 1), not (1,)
scale_const = torch.full((1, 1), 1.0 / (dim ** 0.5), dtype=torch.float32)
scale_load = OffChipLoad(
    underlying=scale_const,
    stride=(0,),
    out_shape_tiled=(1,),
    tile_row=1, tile_col=1, par_dispatch=1,
)
# Stream: shape=(1, 1), dtype=Tile(1, 1)
```

### OffChipStore — Store tiles to HBM (sink, no output stream)
```python
OffChipStore(
    graph=MultiDiGraph,
    input=StepOps,
    par_dispatch=int,
    store_file_name="output",      # Name for simulation output files
)
```

### RandomOffChipLoad — Load tiles from HBM using address stream (indirect access)
```python
RandomOffChipLoad(
    graph=MultiDiGraph,
    underlying=torch.Tensor,       # Source tensor (HBM)
    raddr=StepOps,                 # Address stream — each element is a (1,1) Uint64 tile
    tile_row=int,                  # Tile height
    tile_col=int,                  # Tile width
    base_addr_byte=int,            # Base address offset in bytes
    par_dispatch=int,              # Parallel tile transfers
    mock_bf16=False,
)
```
**Shape rule:** Output stream shape = `raddr.shape`, dtype = `Tile(tile_row, tile_col)`.

### RandomOffChipStore — Store tiles to HBM using address stream (indirect access)
```python
RandomOffChipStore(
    graph=MultiDiGraph,
    underlying=torch.Tensor,       # Destination tensor shape reference
    wdata=StepOps,                 # Data stream to write
    waddr=StepOps,                 # Address stream — (1,1) Uint64 tiles
    tile_row=int,                  # Tile height
    tile_col=int,                  # Tile width
    base_addr_byte=int,            # Base address offset in bytes
    par_dispatch=int,              # Parallel tile transfers
    ack_based_on_waddr=True,       # Ack shape based on waddr (True) or wdata (False)
    buffer_depth=1,                # Store buffer depth
    mock_bf16=False,
)
```
**Shape rule:** Output stream (ack) shape = `waddr.shape` (if ack_based_on_waddr) or `wdata.shape`, dtype = `Bool`.
**Use case:** Writing results back to HBM at dynamically-computed addresses (e.g., KV cache updates).

### DynOffChipLoad — Dynamic off-chip load with ref stream
```python
DynOffChipLoad(
    graph=MultiDiGraph,
    ref=StepOps,                   # Reference stream (determines outer shape)
    underlying=torch.Tensor,       # Source tensor
    stride=(s0, s1, ...),          # Stride per inner dimension
    out_shape_tiled=(d0, d1, ...), # Inner tiled shape appended to ref shape
    tile_row=int,                  # Tile height
    tile_col=int,                  # Tile width
    par_dispatch=int,              # Parallel tile transfers
    mock_bf16=False,
)
```
**Shape rule:** Output stream shape = `ref.shape + out_shape_tiled`, dtype = `Tile(tile_row, tile_col)`.
**Use case:** Loading tiles whose outer iteration is driven by a dynamic ref stream (e.g., loading per-token data where the number of tokens is dynamic).

## Generator Operators (from utility_ops)

### SelectGen — Generate selector stream from tensor
```python
SelectGen(is_multihot=bool, tensor=torch.Tensor, n=int)
```
**Shape rule:** Output shape = `(1,) + tensor.shape[:-1]`.
**Dtype:** `MultiHot(n)` if `is_multihot=True`, else `Index(n)`.

### ExpertAddrGen — Generate expert weight addresses
```python
ExpertAddrGen(
    graph=MultiDiGraph,
    input=StepOps,                 # Input stream (drives iteration)
    num_tile_per_expert=int,       # Number of tiles per expert
    expert_addr_base=int,          # Base address for expert weights
)
```
**Shape rule:** Output shape = `input.shape + (num_tile_per_expert, 1)`, dtype = `Tile(Uint64(), (1,1))`.

**Example:**
```python
# merged_data: (DynDim,) — merged expert token stream
addr_gen = ExpertAddrGen(
    graph=graph, input=merged_data,
    num_tile_per_expert=D // tile_N,  # tiles per expert
    expert_addr_base=D * N * 4,       # byte offset between experts
)
# addr_gen: shape=(DynDim, D//tile_N, 1), dtype=Tile(Uint64, (1,1))
```

### MetadataGen — Generate metadata stream from tensor
```python
MetadataGen(tensor=torch.Tensor)
```
**Shape rule:** Output shape = `(1,) + tensor.shape`, dtype = `Tile(Uint64(), (1,1))`.

### CacheReadAddrGen — Generate KV cache read addresses
```python
CacheReadAddrGen(
    graph=MultiDiGraph,
    idx=StepOps,                   # Token/batch index stream
    seq_len=StepOps,               # Sequence length stream
    row_offset=int,                # Row offset in the KV cache
)
```
**Shape rule:** Output shape = `idx.shape + (DynDim,)`, dtype = `Tile(Uint64(), (1,1))`.

### FilterLastTile — Filter stream for last-tile detection
```python
FilterLastTile(graph=MultiDiGraph, input=StepOps)
```
**Shape rule:** Output shape = `input.shape + (DynDim,)`, dtype = `MultiHot(2)`.

## Compute Operators

**Compute bandwidth budget:** The chip has a total compute bandwidth of **{total_compute_bw}**. Each compute operator declares a `compute_bw` parameter that allocates a share of this total. The sum of all `compute_bw` values across all compute operators in the graph must not exceed {total_compute_bw}. Ideally the full budget is used (sum == {total_compute_bw}). Allocating more bandwidth to an operator increases its throughput; distribute the budget based on which operators are on the critical path.

### BinaryMap — Element-wise binary operation
```python
BinaryMap(graph, in1, in2, fn=map_fn.X(), write_back_mu=bool, compute_bw=int)
```
**Shape rule:** Output shape = `in1.shape`. Both inputs must have matching static dimensions.
**Tile dtype:** Result of `fn.apply(in1_tile, in2_tile)` — e.g., Div on `Tile(64,128)` / `Tile(64,1)` → `Tile(64,128)`.

### UnaryMap — Element-wise unary operation
```python
UnaryMap(graph, input, fn=map_fn.X(), write_back_mu=bool, compute_bw=int)
```
**Shape rule:** Output shape = input shape.
**Tile dtype:** Result of `fn.apply(input_tile)` — e.g., RowWiseSum on `Tile(64,64)` → `Tile(64,1)`.

### BinaryMapAccum — Binary op with reduction (GEMM)
```python
BinaryMapAccum(graph, in1, in2, fn, init_fn, rank, write_back_mu, compute_bw)
```
**Shape rule:** Output shape = `in1.shape[:-rank]`. Removes the last `rank` dimensions (reduction).
**CRITICAL:** `in1.shape == in2.shape` — both inputs must have identical shapes.
**Tile dtype:** Result of `fn.apply(in1_tile, in2_tile)` — e.g., Matmul on `Tile(tm,tk)` @ `Tile(tk,tn)` → `Tile(tm,tn)`.
**init_fn shape** must match the output tile shape.

**Example — GEMM with K-reduction:**
```python
# in1: shape=(1, M//tm, N//tn, K//tk), tile=Tile(tm, tk)
# in2: shape=(1, M//tm, N//tn, K//tk), tile=Tile(tk, tn)
# rank=1 removes last dim (K//tk)
matmul = BinaryMapAccum(graph, in1=x_load, in2=w_load,
    fn=map_accum_fn.Matmul(),
    init_fn=init_fn.Zero(shape=(tile_m, tile_n), dtype=Float32()),
    rank=1, write_back_mu=False, compute_bw=4096)
# Output: shape=(M//tm, N//tn), tile=Tile(tm, tn)
```

### Accum — Reduction/accumulation
```python
Accum(graph, input, output_stream_dtype, fn, init_fn, accum_rank, write_back_mu, compute_bw)
```
**Shape rule:** Output shape = `input.shape[:-accum_rank]`. Removes the last `accum_rank` dimensions.
**Tile dtype:** Explicitly provided via `output_stream_dtype` parameter (not inferred).

**Example — sum over seq_k tiles:**
```python
# input: shape=(outer, seq_q, seq_k), tile=Tile(64, 64)
row_sum = Accum(graph, input=exp_qkt,
    output_stream_dtype=Tile(tile_dtype=Float32(), shape=(64, 64)),
    fn=accum_fn.Add(),
    init_fn=init_fn.Zero(shape=(64, 64), dtype=Float32()),
    accum_rank=1, write_back_mu=False, compute_bw=4096)
# Output: shape=(outer, seq_q), tile=Tile(64, 64)
```

## Shape Transformation Operators

### Promote — Insert size-1 dimension
```python
Promote(graph, input, promote_rank=int)
```
**Shape rule:** Inserts a `1` at position `len(shape) - promote_rank` from the left.

**Example:**
```python
# input shape: (2, 3)
# promote_rank=1 → insert 1 at position len-1 = 1 → (2, 1, 3)
# promote_rank=2 → insert 1 at position len-2 = 0 → (1, 2, 3)
```

### ExpandRef — Expand size-1 dims to match reference
```python
ExpandRef(graph, input, ref, expand_rank=int)
```
**Shape rule:** Output shape = `ref.shape`. The last `expand_rank` dims of input must be `1`.
**Constraint:** `input.shape[:-expand_rank] == ref.shape[:-expand_rank]` (outer dims match).

### RepeatStatic — Append a new innermost dimension
```python
RepeatStatic(graph, input, repeat_factor=int)
```
**Shape rule:** Output shape = `input.shape + (repeat_factor,)`. Adds one dimension.

### Flatten — Merge adjacent dimensions
```python
Flatten(graph, input, min_rank=int, max_rank=int)
```
**Shape rule:** Merges dimensions at rank positions `min_rank` through `max_rank` (counting from the right, 0-indexed) into a single dimension whose size is the product of the merged sizes.

### Reshape — Split a dimension into two
```python
Reshape(graph, input, chunk_size=int, reshape_rank=int, write_back_mu=bool, add_outer_dim=False, pad_fn=None)
```
**Shape rule:** Splits dimension at `rank - reshape_rank` into `(ceil(dim/chunk_size), chunk_size)`.
**Constraint:** If `reshape_rank != 0`, the dimension must be evenly divisible by `chunk_size`.

## Data Reuse Operators

### Bufferize — Capture stream into on-chip ring buffer
```python
Bufferize(graph, input, rank=int, off_chip=False)
```
**Shape rule:** Output shape = `input.shape[:-rank]`. The last `rank` dimensions are absorbed into a Buffer.
**Dtype change:** `Tile(r,c)` → `Buffer(buff_dtype=Tile(r,c), shape=input.shape[-rank:])`.
**Constraint:** Input must be Tile dtype. `rank` must be >= 1 (rank=0 is invalid — there must be at least one dimension to buffer).

### Streamify — Read buffer back as a stream
```python
Streamify(graph, input, repeat_factor=[int, ...], rank=int, off_chip=False)
```
**Shape rule:** Output shape = `input.shape + tuple(repeat_factor) + buffer_shape`.
Where `buffer_shape` = the shape stored in the Buffer dtype.
**Dtype change:** `Buffer(Tile(r,c), shape=(...))` → `Tile(r,c)` (unwraps the buffer).
**Constraint:** Input must be Buffer dtype (output of Bufferize).

**CRITICAL — Bufferize/Streamify shape math:**

The output of Streamify must match the shape of the other stream it will be paired with.
Here is the shape flow:

```
Input stream:  shape = (1, d0, d1, ..., dN)
                                    ^^^^^^^ last `rank` dims
Bufferize(rank=R):
  Output:      shape = (1, d0, ..., d(N-R))
               dtype = Buffer(Tile, shape=(d(N-R+1), ..., dN))

Streamify(repeat_factor=[r0, r1, ...]):
  Output:      shape = (1, d0, ..., d(N-R)) + (r0, r1, ...) + (d(N-R+1), ..., dN)
               dtype = Tile (unwrapped)
```

To get the **same shape as the original stream**, use `repeat_factor=[]` (empty list).
To **add new dimensions** for broadcasting, include them in `repeat_factor`.

**Example — WRONG (shape mismatch):**
```python
# x_load: shape=(1, 2, 2, 2), tile=Tile(128,128)
x_buf = Bufferize(graph, x_load, rank=2)
# x_buf: shape=(1, 2), dtype=Buffer(Tile(128,128), shape=(2, 2))

x_stream = Streamify(graph, x_buf, repeat_factor=[2], rank=2)
# x_stream: shape=(1, 2) + (2,) + (2, 2) = (1, 2, 2, 2, 2) ← RANK 4!
# w_load:   shape=(1, 2, 2, 2)                                ← RANK 3
# BinaryMapAccum FAILS: shapes don't match!
```

**Example — CORRECT (buffer one dim, replay for reuse):**
```python
# Goal: buffer weight tiles and replay them for each M-tile.
# w_load: shape=(1, M//tm, N//tn, K//tk), tile=Tile(tk, tn)
# We want to buffer over (N//tn, K//tk) and replay for each M-tile.

w_buf = Bufferize(graph, w_load, rank=2, off_chip=False)
# w_buf: shape=(1, M//tm), dtype=Buffer(Tile(tk,tn), shape=(N//tn, K//tk))

w_stream = Streamify(graph, w_buf, repeat_factor=[], rank=2, off_chip=False)
# w_stream: shape=(1, M//tm) + () + (N//tn, K//tk) = (1, M//tm, N//tn, K//tk)
# Now matches x_load shape — BinaryMapAccum works!
```

### DynStreamify — Dynamic streamify from buffer
```python
DynStreamify(graph, input, ref, repeat_rank=int, bufferized_rank=int, off_chip=False)
```
**Shape rule:** Output shape = `ref.shape + buffer_shape`.
Output dtype = unwrapped Tile from Buffer.
**Constraint:** Input must be Buffer dtype. Last `repeat_rank + 1` dims of input must be `1`.

## Multi-Consumer Operators

### Broadcast — Replicate stream for multiple consumers
```python
Broadcast(graph, input, num_consumers=int)
```
**Shape rule:** Each output has same shape as input. Creates `num_consumers` copies.
**Access:** `(broadcast, 0)`, `(broadcast, 1)`, etc.

### Parallelize — Split stream across parallel consumers
```python
Parallelize(graph, input, parallelize_rank=int, num_consumers=int, switch_cycles=None, write_back_mu=False)
```
**Shape rule:** Divides outermost dimension by `num_consumers`. Each output: `(shape[0]//num_consumers,) + shape[1:]`.
**Constraint:** `parallelize_rank == input.rank`. Outermost dim must be divisible by `num_consumers`.
**Access:** `(parallelize, 0)`, `(parallelize, 1)`, etc.

### EagerMerge — Merge multiple streams
```python
EagerMerge(graph, inputs=[op0, op1, ...], input_rank=int)
```
**Shape rule:** Data output: `(sum_of_dim0s,) + inputs[0].shape[1:]` — concatenates outermost dim.
Also produces select output: `(sum_of_dim0s,)`, dtype=MultiHot.
**Access:** `eager_merge.data_tuple()`, `eager_merge.select_tuple()`.

### FlatPartition — Dynamic routing to N consumers (MoE)
```python
FlatPartition(graph, input, control, partition_rank=int, switch_cycles=[int,...], write_back_mu=bool, num_consumers=int)
```
**Shape rule:** Each output: `(DynDim,) + input.shape[-partition_rank:]` — outermost becomes dynamic.
**Access:** `(flat_partition, 0)`, `(flat_partition, 1)`, etc.

### FlatReassemble — Reassemble partitioned streams
```python
FlatReassemble(graph, inputs=[...], control, reassemble_rank=int, switch_cycles=[int,...], write_back_mu=bool)
```
**Shape rule:** Output shape = `control.shape + (DynDim,) + inputs[0].shape[-reassemble_rank:]`.

### RetileStreamify — Split tiles along row or column
```python
RetileStreamify(graph, input, split_row=bool, filter_mask=False)
```
**Shape rule (split_row=True):** Last dim becomes `last_dim * tile_rows`. Tile becomes `Tile(1, tile_cols)`.
**Shape rule (split_row=False):** Last dim becomes `last_dim * tile_cols`. Tile becomes `Tile(tile_rows, 1)`.
If `filter_mask=True`, last dim becomes DynDim.

## Available Functions

### map_fn (for BinaryMap, UnaryMap)
- `map_fn.Matmul(weight_transposed=False)` — `Tile(M,K) @ Tile(K,N) → Tile(M,N)`
- `map_fn.Add()` — element-wise add, broadcasts `Tile(1,N)` or `Tile(M,1)` operands
- `map_fn.Mul()` — element-wise multiply, broadcasts `Tile(1,N)` or `Tile(M,1)` operands
- `map_fn.Div()` — element-wise divide, broadcasts `Tile(M,1)` denominator
- `map_fn.Silu()` — SiLU activation, same tile shape (unary)
- `map_fn.Exp()` — exponential, same tile shape (unary)
- `map_fn.RowWiseSum()` — `Tile(M,N) → Tile(M,1)` (reduce columns, unary)
- `map_fn.RowWiseAppend()` — append row to tile: `(Tile(N,D), Tile(1,D)) → Tile(N,D)` (binary)
- `map_fn.SetOffset()` — set tile offset: `(Tile(M,N), Tile(1,1)) → Tile(M,N)` (binary)
- `map_fn.CacheWriteAddrGen(row_offset=int)` — generate cache write address: `(Tile(Uint64,1,1), Tile(Uint64,1,1)) → Tile(Uint64,1,1)` (binary)
- `map_fn.DynMatmul(weight_transposed=False)` — dynamic shape matmul, supports DynTile inputs

### map_accum_fn (for BinaryMapAccum)
- `map_accum_fn.Matmul(weight_transposed=False)` — `Tile(M,K) @ Tile(K,N) → Tile(M,N)`
- `map_accum_fn.Matmul(weight_transposed=True)` — `Tile(M,K) @ Tile(N,K)^T → Tile(M,N)`
- `map_accum_fn.DynMatmul(weight_transposed=False)` — dynamic matmul with accumulation

### accum_fn (for Accum)
- `accum_fn.Add()` — sum tiles element-wise
- `accum_fn.Mul()` — multiply tiles element-wise
- `accum_fn.RetileRow()` — retile along rows: accumulates `Tile(1,D)` rows into `Tile(N,D)` tile
- `accum_fn.SignalReqAllRead()` — signal for all-read synchronization, outputs `Tile(Uint64, (1,1))`

### init_fn (for BinaryMapAccum, Accum)
- `init_fn.Zero(shape=(rows, cols), dtype=Float32())` — initialize to zeros
- `init_fn.Empty(shape=(rows, cols), dtype=Float32())` — uninitialized (use when first write overwrites entirely)

### Data Types

**Element types (for Tile dtype):**
- `Float32()` — 32-bit float (4 bytes)
- `Float16()` — 16-bit float (2 bytes)
- `Uint32()` — 32-bit unsigned integer (4 bytes)
- `Uint64()` — 64-bit unsigned integer (8 bytes, used for addresses)
- `Bool()` — boolean (1 byte, used for ack streams)

**Stream dtype types:**
- `Tile(tile_dtype=ElementTP, shape=(rows, cols))` — fixed-size tile
- `DynTile(tile_dtype=ElementTP, shape=(rows_or_DynDim, cols_or_DynDim))` — tile with dynamic dimensions
- `Buffer(buff_dtype=Tile, shape=(...))` — on-chip buffer holding tiles (output of Bufferize)
- `MultiHot(total_n=int)` — multi-hot selector (used by SelectGen, FlatPartition)
- `Index(active_n=int)` — index selector

**Import locations:**

## Composition Patterns

These are complete, verified patterns from real working STeP IR baselines.

### Pattern 1 — Weight-Stationary GEMM (from gemm.py)

Complete `build_graph()` for C = A @ B with inline shape annotations.

```python
def build_graph(dims):
    M, K, N = dims["M"], dims["K"], dims["N"]
    tile_m, tile_k, tile_n = 128, 128, 128
    par_dispatch = 4
    compute_bw = 4096

    graph = MultiDiGraph()
    A = torch.randn(M, K, dtype=torch.float32)
    B = torch.randn(K, N, dtype=torch.float32)

    # A is (M, K). Tiled stream iterates (M-tiles, N-tiles, K-tiles).
    # stride: M-dim advances K//tile_k tiles, N-dim broadcast (0), K-dim advances 1.
    a_load = OffChipLoad(
        underlying=A,
        stride=(K // tile_k, 0, 1),
        out_shape_tiled=(M // tile_m, N // tile_n, K // tile_k),
        tile_row=tile_m, tile_col=tile_k, par_dispatch=par_dispatch,
    )
    # a_load: shape=(1, M//128, N//128, K//128), tile=Tile(128, 128)

    # B is (K, N). Same iteration order, different strides.
    # stride: M-dim broadcast (0), N-dim advances 1, K-dim advances N//tile_n.
    b_load = OffChipLoad(
        underlying=B,
        stride=(0, 1, N // tile_n),
        out_shape_tiled=(M // tile_m, N // tile_n, K // tile_k),
        tile_row=tile_k, tile_col=tile_n, par_dispatch=par_dispatch,
    )
    # b_load: shape=(1, M//128, N//128, K//128), tile=Tile(128, 128)

    # Matmul: Tile(128,128) @ Tile(128,128) -> Tile(128,128), rank=1 reduces K-dim
    matmul = BinaryMapAccum(
        graph=graph, in1=a_load, in2=b_load,
        fn=map_accum_fn.Matmul(),
        init_fn=init_fn.Zero(shape=(tile_m, tile_n), dtype=Float32()),
        rank=1, write_back_mu=True, compute_bw=compute_bw,
    )
    # matmul: shape=(1, M//128, N//128), tile=Tile(128, 128)

    output_op = OffChipStore(graph=graph, input=matmul,
                             par_dispatch=par_dispatch, store_file_name="output")
    graph = infer_broadcast(graph)
    return graph, output_op
```

### Pattern 2 — On-chip Reuse via Bufferize/Streamify (from rmsnorm_matmul.py)

The most common optimization pattern AND the biggest source of bugs.
This shows chaining an intermediate result (normalized) into a matmul
without storing it back to HBM, using Bufferize→Streamify→BinaryMapAccum.

```python
# Context: tile_m=64, tile_n=64, K=full width of row
# normalized: shape=(1, M_tiles, 1), tile=Tile(tile_m, K)
#   (each tile holds a full row of K elements)

# Step 1: Bufferize — absorb the trailing dim (size 1) into on-chip buffer
buff = Bufferize(graph, normalized, rank=1)
# buff: shape=(1, M_tiles), dtype=Buffer(Tile(tile_m, K), shape=(1,))
#   The last 1 dim was absorbed. Each buffer holds 1 tile of (tile_m, K).

# Step 2: Streamify — replay for each N-tile column position
norm_expanded = Streamify(graph, buff, repeat_factor=[N_tiles], rank=1)
# norm_expanded: shape=(1, M_tiles, N_tiles, 1), tile=Tile(tile_m, K)
#   Shape math: (M_tiles,) + (N_tiles,) + (1,) = (M_tiles, N_tiles, 1)
#   The N_tiles repeat means each buffered tile is emitted N_tiles times.

# Step 3: Load weights with matching shape
w_load = OffChipLoad(
    underlying=w_t,
    stride=(0, 1, 0),
    out_shape_tiled=(M_tiles, N_tiles, 1),  # K_tiles=1 (full K in one tile)
    tile_row=K, tile_col=tile_n, par_dispatch=par_dispatch,
)
# w_load: shape=(1, M_tiles, N_tiles, 1), tile=Tile(K, tile_n)
# CRITICAL: norm_expanded and w_load now have the same stream shape!

# Step 4: Matmul — Tile(tile_m, K) @ Tile(K, tile_n) -> Tile(tile_m, tile_n)
matmul = BinaryMapAccum(
    graph=graph, in1=norm_expanded, in2=w_load,
    fn=map_accum_fn.Matmul(),
    init_fn=init_fn.Zero(shape=(tile_m, tile_n), dtype=Float32()),
    rank=1, write_back_mu=True, compute_bw=2048,
)
# matmul: shape=(1, M_tiles, N_tiles), tile=Tile(tile_m, tile_n)
```

**Shape math recap:**
```
normalized           (M_tiles, 1)        Tile(tile_m, K)
  -> Bufferize(rank=1)
buff                 (M_tiles,)          Buffer(Tile(tile_m, K), shape=(1,))
  -> Streamify(repeat_factor=[N_tiles], rank=1)
norm_expanded        (M_tiles, N_tiles, 1)  Tile(tile_m, K)
w_load               (M_tiles, N_tiles, 1)  Tile(K, tile_n)    <-- must match!
  -> BinaryMapAccum(rank=1)
matmul               (M_tiles, N_tiles)     Tile(tile_m, tile_n)
```

### Pattern 3 — Softmax with Broadcast (from bmm_softmax.py)

Shows Broadcast→RowWiseSum→Div chain with compute_bw budget distribution.
Full-N tiles mean the softmax row reduction stays within a single tile.

```python
# Context: tile_m=64, N=full width. Total compute_bw budget = 4096.
# exp_x: shape=(1, B, M_tiles, 1), tile=Tile(tile_m, N)
#   (after Exp on matmul output, compute_bw=512)

# Broadcast to feed exp_x to both RowWiseSum and Div
exp_bc = Broadcast(graph=graph, input=exp_x, num_consumers=2)
# (exp_bc, 0) and (exp_bc, 1) each: shape=(1, B, M_tiles, 1), tile=Tile(tile_m, N)

# RowWiseSum: reduce columns within each tile
rowsum = UnaryMap(
    graph=graph, input=(exp_bc, 1),
    fn=map_fn.RowWiseSum(), write_back_mu=False, compute_bw=512,
)
# rowsum: shape=(1, B, M_tiles, 1), tile=Tile(tile_m, 1)

# Div: each row divided by its sum (Tile broadcasts (tile_m,1) denom)
softmax_out = BinaryMap(
    graph=graph, in1=(exp_bc, 0), in2=rowsum,
    fn=map_fn.Div(), write_back_mu=True, compute_bw=1024,
)
# softmax_out: shape=(1, B, M_tiles, 1), tile=Tile(tile_m, N)

# Budget breakdown: matmul=2048 + exp=512 + rowsum=512 + div=1024 = 4096
```

## Common Mistakes

### Mistake 1 — Bufferize/Streamify shape mismatch

Wrong `repeat_factor` producing a rank mismatch with the BinaryMapAccum partner.

**WRONG:**
```python
# normalized: shape=(1, M_tiles, 1), tile=Tile(tile_m, K)
buff = Bufferize(graph, normalized, rank=1)
# buff: shape=(1, M_tiles), Buffer(Tile(tile_m, K), shape=(1,))

norm_stream = Streamify(graph, buff, repeat_factor=[N_tiles, 1], rank=1)
# norm_stream: shape=(1, M_tiles, N_tiles, 1, 1)  <-- RANK 4!
# w_load:      shape=(1, M_tiles, N_tiles, 1)      <-- RANK 3
# BinaryMapAccum FAILS: shapes (M_tiles, N_tiles, 1, 1) != (M_tiles, N_tiles, 1)
```

**RIGHT:**
```python
norm_stream = Streamify(graph, buff, repeat_factor=[N_tiles], rank=1)
# norm_stream: shape=(1, M_tiles, N_tiles, 1)  <-- matches w_load
```

**Rule:** `repeat_factor` inserts new dims *between* the outer shape and the
buffered shape. The resulting shape is `outer + repeat_factor + buffer_shape`.
Only add repeat dims that actually correspond to iteration dimensions you need.

### Mistake 2 — OffChipLoad strides producing mismatched shapes

Both loads for a BinaryMapAccum must produce the same `out_shape_tiled`.
A common error is using different `out_shape_tiled` values because the stride
was computed for the wrong tiling.

**WRONG:**
```python
# A is (M, K), B is (K, N). Want weight-stationary GEMM.
a_load = OffChipLoad(
    underlying=A,
    stride=(K // tile_k, 0, 1),
    out_shape_tiled=(M // tile_m, N // tile_n, K // tile_k),
    tile_row=tile_m, tile_col=tile_k, par_dispatch=4,
)
# a_load: shape=(1, M//tm, N//tn, K//tk)

b_load = OffChipLoad(
    underlying=B,
    stride=(0, 1, K // tile_k),          # <-- stride[2] wrong!
    out_shape_tiled=(N // tile_n, K // tile_k),  # <-- shape wrong!
    tile_row=tile_k, tile_col=tile_n, par_dispatch=4,
)
# b_load: shape=(1, N//tn, K//tk)  <-- RANK 2, doesn't match a_load's RANK 3
```

**RIGHT:**
```python
b_load = OffChipLoad(
    underlying=B,
    stride=(0, 1, N // tile_n),           # M-dim broadcast, N advances 1, K advances N//tn
    out_shape_tiled=(M // tile_m, N // tile_n, K // tile_k),  # same shape as a_load
    tile_row=tile_k, tile_col=tile_n, par_dispatch=4,
)
# b_load: shape=(1, M//tm, N//tn, K//tk)  <-- matches a_load
```

**Rule:** Both inputs to BinaryMapAccum must have identical `out_shape_tiled`.
Use `stride=0` for broadcast dimensions where the same data is reused.

### Mistake 3 — compute_bw budget overflow

Each operator gets its own `compute_bw` allocation, but the total across all
compute operators must not exceed {total_compute_bw} (typically 4096).

**WRONG:**
```python
exp_x = UnaryMap(graph, input=matmul, fn=map_fn.Exp(),
                 write_back_mu=False, compute_bw=4096)          # 4096
rowsum = UnaryMap(graph, input=(exp_bc, 1), fn=map_fn.RowWiseSum(),
                  write_back_mu=False, compute_bw=4096)         # + 4096
softmax = BinaryMap(graph, in1=(exp_bc, 0), in2=rowsum,
                    fn=map_fn.Div(), write_back_mu=True, compute_bw=4096)  # + 4096
# Total = 12288 — EXCEEDS budget of 4096!
```

**RIGHT:**
```python
exp_x = UnaryMap(graph, input=matmul, fn=map_fn.Exp(),
                 write_back_mu=False, compute_bw=512)           # 512
rowsum = UnaryMap(graph, input=(exp_bc, 1), fn=map_fn.RowWiseSum(),
                  write_back_mu=False, compute_bw=512)          # + 512
softmax = BinaryMap(graph, in1=(exp_bc, 0), in2=rowsum,
                    fn=map_fn.Div(), write_back_mu=True, compute_bw=1024)  # + 1024
# Total = 2048 — leaves 2048 for matmul upstream. Full budget used.
```

**Rule:** Distribute the budget based on which operators are on the critical
path. Matmul typically gets the largest share; lightweight ops (Exp, RowWiseSum)
get less.
