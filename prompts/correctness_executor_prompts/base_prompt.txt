# STeP Graph Construction Rules

A STeP kernel is a Python function `build_graph()` that returns `(graph, output_op)` where:
- `graph` is a `networkx.MultiDiGraph`
- `output_op` is the final `OffChipStore` operation

**You only need to output `build_graph()`.** Imports, constants (M, K, N, SEED, etc.), and `compute_gold()` are provided externally. Your `build_graph()` can reference these module-level constants directly.

**Your primary goal is CORRECTNESS.** The kernel must:
1. Compile without errors
2. Run the simulation to completion (produce output files)
3. Produce output that matches the PyTorch reference (torch.allclose with rtol=1e-4, atol=1e-6)

Do NOT worry about performance optimization. A slow correct kernel is the goal.

## Required Imports

import torch
from networkx import MultiDiGraph

## Rules
1. Every graph must have at least one OffChipLoad and one OffChipStore.
2. Use `write_back_mu=False` for intermediate results. Use `write_back_mu=True` on the **last compute node** feeding OffChipStore.
3. Tile dimensions must evenly divide tensor dimensions.
4. `build_graph()` must return `(graph, final_store_op)`.
5. Do not use float16 unless explicitly required.
6. Always call `graph = infer_broadcast(graph)` before returning.
7. Both inputs to BinaryMapAccum must have identical stream shapes.
8. **Compute bandwidth budget:** Total is {total_compute_bw}. Sum of all `compute_bw` values must not exceed {total_compute_bw}.
9. **HBM configuration:** {hbm_channel_num} channels.
10. **On-chip scratchpad:** {on_chip_memory_human} ({on_chip_memory_bytes} bytes). Peak usage must not exceed this.

## Common Correctness Pitfalls
- **Stream shape mismatches:** Both inputs to BinaryMap and BinaryMapAccum must have the same stream shape. Verify shapes at every step.
- **Wrong OffChipLoad strides:** A stride of 0 means broadcast (same data reused). A stride of N means advance N tiles per step. Getting this wrong loads garbage data.
- **RNG seeding:** The generated code must call `torch.manual_seed(SEED)` and create tensors in the same order as `compute_gold()` to match the reference output.
- **nn.Linear weight layout:** `nn.Linear(in, out)` stores weight as `(out, in)`. To get `(in, out)` for matmul, use `weight.T`.
- **OffChipStore tile count:** The stream feeding OffChipStore must produce exactly the right number of tiles to fill the output tensor.
- **Bufferize/Streamify misuse:** Do NOT wrap BinaryMapAccum inputs in Bufferize/Streamify. It corrupts the accumulation.
