# STeP Graph Construction Rules

A STeP kernel is a Python function `build_graph()` that returns `(graph, output_op)` where:
- `graph` is a `networkx.MultiDiGraph`
- `output_op` is the final `OffChipStore` operation

**You only need to output `build_graph()`.** Imports, constants (M, K, N, SEED, etc.), and `compute_gold()` are provided externally. Your `build_graph()` can reference these module-level constants directly.

## Required Imports
```python
import torch
from networkx import MultiDiGraph
from step_py.ops import (
    OffChipLoad, OffChipStore, BinaryMap, UnaryMap, BinaryMapAccum,
    Accum, Flatten, Reshape, Promote, ExpandRef,
    Bufferize, Streamify, DynStreamify, RetileStreamify,
    FlatPartition, FlatReassemble, Broadcast, RepeatStatic,
    EagerMerge, Parallelize,
)
from step_py.utility_ops import SelectGen, FilterLastTile, ConsumerContext
from step_py.functions import map_fn, map_accum_fn, accum_fn, init_fn
from step_py.datatype import Tile, DynTile, Stream, Float16, Float32, Uint32, Uint64, Bool, MultiHot, Index, Select
from rewrite.broadcast import infer_broadcast
```

## Operator Quick Reference
- `OffChipLoad` / `OffChipStore` ‚Äî HBM ‚Üî on-chip tile transfer
- `BinaryMap(fn)` ‚Äî element-wise binary op (Matmul, Add, Mul, Div)
- `UnaryMap(fn)` ‚Äî element-wise unary op (Silu, Exp, RowWiseSum)
- `BinaryMapAccum(fn, init_fn, rank)` ‚Äî fused map-accumulate (GEMM). Removes last `rank` dims.
- `Accum(fn, init_fn, accum_rank)` ‚Äî reduction. Removes last `accum_rank` dims.
- `Bufferize(rank)` / `Streamify(repeat_factor, rank)` ‚Äî on-chip data reuse. See operator reference for shape rules.
- `Broadcast(num_consumers)` ‚Äî replicate stream. Access: `(broadcast, 0)`.
- `Promote(promote_rank)` ‚Äî insert size-1 dimension.
- `ExpandRef(ref, expand_rank)` ‚Äî expand size-1 dims to match reference.
- `RepeatStatic(repeat_factor)` ‚Äî append new innermost dimension.
- `Flatten(min_rank, max_rank)` / `Reshape(chunk_size, reshape_rank)` ‚Äî shape transforms.
- `Parallelize` / `EagerMerge` / `FlatPartition` / `FlatReassemble` ‚Äî multi-consumer/MoE ops.

## Rules
1. Every graph must have at least one OffChipLoad and one OffChipStore.
2. Use `write_back_mu=False` for intermediate results to avoid unnecessary HBM round-trips.
3. Tile dimensions must evenly divide tensor dimensions (or pad).
4. `build_graph()` must return `(graph, final_store_op)`.
5. Do not use float16 unless the baseline uses it.
6. Always call `graph = infer_broadcast(graph)` before returning.
7. Both inputs to BinaryMapAccum must have identical stream shapes.
